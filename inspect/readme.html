Inspect.js

Lightning-fast, modular form validation with 50+ built-in rules.
Zero dependencies, works with any UI framework.

v2.0

Quick start

Inspect.js provides a declarative validation engine. Define rules, attach to any form, and get powerful validation with instant feedback. Supports nested fields, file validation, multi-selects, checkboxes, radio groups, and dynamic data.

// 1. Define rules const rules = [ { field: 'email', pretty: 'Email', rules: { required: true, email: true, maxlen: 100 } }, { field: 'age', pretty: 'Age', rules: { required: true, min: 18, max: 99 } }, { field: 'skills[]', pretty: 'Skills', rules: { required: true, min_elem: 1 } } ]; // 2. Initialize validator const inspector = new Inspect().init(document.getElementById('myForm'), rules); // 3. Validate on submit form.addEventListener('submit', async (e) => { e.preventDefault(); const result = await inspector.validate(); if (result.valid) console.log('Data:', result.data); });

✨ Supports array fields (name="hobbies[]"), file inputs, custom error messages, and dynamic rules.

API Reference

new Inspect(options) → .init(formElement, rules)

Creates validator instance bound to a specific form. Returns an object with core methods:

  • ⚡ validate() → { valid, errors, data } — runs validation against current form, shows errors automatically.
  • ⚡ validateData(dataObject) → same result but validates raw JS object (useful for API testing).
  • ⚡ parseFormData() → extracts form data respecting array keys, file lists, multi-values.
  • ⚡ clearErrors() → removes all validation error styles and messages.

Rule definition structure

{ field: 'username', // form field name (supports "name[]" for arrays) pretty: 'Username', // display name in error messages rules: { required: true, minlen: 3, maxlen: 30, alpha_num: true }, messages: { // optional custom messages required: 'Username is mandatory.', minlen: 'Too short, need at least 3 chars' } }

The rules object supports any built-in validator (see table below) and also custom validators by extending Inspect.validators.

Custom error handling & messages

Override default messages or provide a global error handler:

const customInspector = new Inspect({ messages: { required: '⚠️ {1} cannot be blank.', email: '✉️ Invalid email address.' }, errorHandler: (field, message, config, show) => { // Custom UI logic (toast, tooltip, etc) if (show) field.classList.add('custom-error'); else field.classList.remove('custom-error'); } }).init(form, rules);

You can also define field-specific messages inside the rule definition (messages: { min: 'Age must be at least {0}' }).
